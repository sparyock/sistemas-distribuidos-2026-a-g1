02 — Architecture Decision Records
ADR-001 · Generic
Base Service
El patrón más crítico del sprint — eliminación de código duplicado.
Proceso aplicado
1
Identificación de patrones
Repository, DTO, Service Layer ya implementados.
2
Análisis del proyecto
Detectar qué ADRs faltan: Generic Service, MapStruct, Exceptions.
3
Identificación de antipatrones
Copy-Paste Programming, Magic Strings, lógica CRUD repetida.
4
Detección de malas prácticas
Mapeo manual con mapToDTO() y RuntimeException genérico.
5
Diseño e implementación
IBaseService<T,D> + clase abstracta BaseServiceImpl.
6
Análisis de impacto
~40% menos código, mayor mantenibilidad y escalabilidad.
❌ Problema
// TurnoServiceImpl
findAll() ~20 lín.
save() ~25 lín.

// UserServiceImpl
// ← mismo código 🔁
Antipatrón Copy-Paste Programming. Si algo cambia, hay que actualizarlo en múltiples lugares.
✅ Solución
IBaseService<T,D>
↑ implements
BaseServiceImpl<T,D>
↑ extends
TurnoSvc
UserSvc
interface IBaseService
<T, D> {
 List<D> findAll();
 D findById(Long id);
 D save(D dto);
 void delete(Long);
}

Prioridad: Alta 🔴

03 — ADRs complementarios
ADRs 002 – 005
Patrones de soporte
4 decisiones que completan la base de calidad del sistema.
ADR-002
🟡 Media prioridad
MapStruct — Mapeo automático
Elimina el método mapToDTO() manual en cada servicio. Con la anotación @Mapper la conversión Entity ↔ DTO se genera automáticamente, eliminando el riesgo de olvidar campos o tener inconsistencias entre servicios.
@Mapper
Entity → DTO
Patrón estructural
ADR-003
🟡 Media prioridad
Excepciones personalizadas
Reemplaza RuntimeException genérico por excepciones específicas del dominio como TurnoNotFoundException. Antipatrón Magic Strings eliminado — mejora el debugging y diferencia claramente los tipos de error.
TurnoNotFoundException
Dominio
Antipatrón Magic Strings
ADR-004
🟢 Baja prioridad
Auditoría de entidades
Registra automáticamente las fechas de creación y modificación en todas las entidades usando Spring Data JPA, sin hacerlo manualmente en cada operación de guardado o actualización del sistema.
@CreatedDate
@LastModifiedDate
Spring Data JPA
ADR-005
🟡 Media prioridad
Bean Validation · @Valid
Garantiza que los datos que llegan a los endpoints sean correctos antes de tocar la capa de negocio. Evita errores en base de datos por datos nulos, vacíos o malformados que de otra forma pasarían sin ser rechazados.
@Valid
@NotNull
@Email
@Size
